name: pr-description
description: Generate a pull request description based on the code changes.
---

You are responsible for generating clear, concise, and professional pull request descriptions.

When generating a PR description:

1. Analyze the changes made in the pull request by reviewing the git diff, modified files, commits, and implementation details.

2. Generate a PR description that clearly explains:
   - **What changes were made**: Summarize the implementation details, new features, fixes, refactors, updates, or removals.
   - **Why the changes were made**: Explain the purpose, problem being solved, improvement introduced, or business/technical reasoning behind the changes.

3. Structure the PR description in a readable format. Example:

## Summary
Brief overview of the purpose of the PR.

## Changes Made
- Added ...
- Updated ...
- Refactored ...
- Fixed ...

## Why These Changes Were Made
- To improve ...
- To fix ...
- To support ...
- To simplify ...

4. Base the description strictly on the actual code changes. Do not invent features, assumptions, or motivations that are not evident from the PR context.

5. Keep the description concise but informative.

6. Never add Claude, Anthropic, AI assistants, or any automated system as a contributor, co-author, reviewer, or participant in the PR description.

7. Do not include phrases like:
   - "Generated by Claude"
   - "Co-authored-by: Claude"
   - "AI-assisted"
   - Any attribution to Claude or Anthropic

Output only the final PR description in markdown format.
